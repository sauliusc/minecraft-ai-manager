⛏ Minecraft AI Manager

The complete admin dashboard and game manager for your Minecraft server — handle players, challenges, events, clans, rewards, and moderation all in one place.

✓ Web Dashboard ✓ Live Events ✓ Clan System ✓ Smart Moderation

🧭 What Is This?

Minecraft AI Manager is a web-based control panel for a Minecraft server. Think of it as the "admin HQ" — a website where server staff can manage everything that happens in the game without having to type long commands in the Minecraft console.

The system has two parts that talk to each other:

🌐

The Web Dashboard

A website that admins and moderators open in their browser to manage the server.

🎮

The Minecraft Plugins

Code running inside the Minecraft server that reacts to what players do in-game.

Whenever something happens in Minecraft (a player joins, breaks a block, files a report), the plugin tells the web dashboard. Whenever an admin does something on the website (creates a challenge, bans a player), the plugin acts on it in-game automatically.

💡

Example: An admin creates a "Mine 100 Diamonds" challenge on the website. The Minecraft plugin automatically starts tracking every player's diamond-mining progress and grants the reward when they hit 100 — no commands needed!

👥 User Roles

There are three types of accounts. Every person on the server has one of these roles:

Role Who Has It What They Can Do
SUPER_ADMIN Server owners / head admins Everything — create challenges, rewards, events, ban players, manage NPCs, see all analytics, configure automatic triggers.
MODERATOR Moderators / helpers View all player data, review reports, mute and kick players, view chat logs and analytics. Cannot ban players or delete core content.
PLAYER Regular players View their own profile and progress, report other players, and use in-game commands (clan, cosmetics, etc.).
ℹ️

Throughout this guide, features marked SUPER_ADMIN are only available to the server owner and head admins. Features marked MODERATOR are available to both mods and admins.

🔑 Logging In

To access the web dashboard, open the server's web address in your browser and you'll see the login screen. Enter your email address and password.

  1. Open the dashboard URL in your browser (your server owner will give you this).

  2. Type your email address and password in the login form.

  3. Click Sign In. You'll go straight to the Dashboard home page.

⚠️

Too many failed attempts: If you type the wrong password 5 times in a row, you'll be locked out for one minute. Wait and try again carefully!

🔐 Staying Logged In

The system uses a short-lived access token (15 minutes) and a longer refresh token (7 days). In plain terms: you stay logged in for up to 7 days without having to sign in again, but the app quietly renews your session in the background. Just use it normally!

💚

🎯

Active Challenges

How many challenges are currently running that players can participate in.

🎁

Rewards Today

How many rewards have been granted to players so far today.

👤 Players

The Players section shows everyone who has ever joined your Minecraft server. Admins and moderators can search for players, view their full history, and take actions.

Access: SUPER_ADMIN MODERATOR

The main players page shows a table of all players with:

  • Username — their Minecraft name
  • Tier — a rank based on how many times they've joined (see below)
  • Last Seen — when they were last online
  • Join Count — total number of logins ever

You can search by username and filter by tier. Results load 20 per page.

🔍 Player Detail Page

Click any player's name to open their full profile. The profile has four tabs:

📊 Overview Tab

Shows the player's basic stats:

  • First Join — when they joined the server for the very first time
  • Last Seen — most recent login date
  • Total Joins — lifetime login count
  • Current Streak — how many days in a row they've logged in
  • Longest Streak — their all-time best streak

Admins also see buttons to Grant Reward and Apply Action (mute/kick/ban).

🎯 Challenge Progress Tab

Shows every challenge the player has started or completed:

  • Challenge name and type
  • Progress bar (e.g. "47 / 100 blocks mined")
  • Status: In Progress or Completed
  • Completion date (if finished)
🎁 Reward History Tab

All rewards the player has ever received:

  • Reward name and type (Item, XP, Currency, etc.)
  • When it was granted
  • Who granted it (admin username or "system" if automatic)
🔨 Moderation History Tab

Any moderation actions taken against this player:

  • Action type: Mute, Kick, Ban, Unmute, Unban
  • Reason given
  • Which admin did it
  • Expiry date (for temporary mutes/bans)

✅ Granting a Reward to a Player

Admins can manually give a player a reward:

  1. Open the player's profile page.

  2. Click the Grant Reward button.

  3. Choose a reward from the dropdown list.

  4. Optionally add a reason (e.g. "Won the build contest").

  5. Click Grant. If the player is online, they get it instantly. If offline, it queues until they log in.

🔨 Applying a Moderation Action

SUPER_ADMIN MODERATOR (Mods can mute/kick only; only Admins can ban)

  1. Open the player's profile page.

  2. Click Apply Action.

  3. Choose the action type: Mute, Kick, or Ban.

  4. Enter a reason (required).

  5. For Mute and Ban, optionally set an expiry date/time (leave blank for permanent).

  6. Click Apply. The action takes effect in-game immediately and is saved to the audit log.

🎯 Challenges

Challenges are tasks that players can complete in Minecraft to earn rewards. Think of them as quests or achievements — "Mine 500 blocks", "Kill 10 skeletons", etc.

📋 Challenge List

The main challenges page shows all challenges with filters:

  • Status filter — Active (running now), Upcoming (starts later), or Expired
  • Type filter — filter by challenge category
  • Difficulty filter — from 1 star (easy) to 5 stars (very hard)

Each challenge card shows the title, type, difficulty stars, and how many players have completed it.

📖 Challenge Types

TypeWhat Players DoExample
BLOCK_BREAKBreak a specific type of block"Mine 100 diamonds"
KILL_MOBKill a specific type of monster"Slay 20 creepers"
CRAFT_ITEMCraft a specific item"Craft 5 golden swords"
TRAVELWalk a certain number of blocks"Travel 10,000 blocks"
CUSTOMAny special plugin-defined action"Win 3 clan wars"

➕ Creating a Challenge SUPER_ADMIN

  1. Go to the Challenges page and click New Challenge.

  2. Enter a title and description.

  3. Choose a type (see table above). Extra config fields appear based on type — for example BLOCK_BREAK asks for the block material and a count target.

  4. Set the difficulty from 1–5 stars.

  5. Set start and end dates for when the challenge is active.

  6. Optionally assign to specific player UUIDs (leave blank to make it available to everyone).

  7. Click Create. The challenge goes live at the start date automatically.

📅 Challenges Calendar

The Calendar view shows all challenges on a monthly grid — green blocks for easy challenges, red blocks for hard ones. If 3 or more hard challenges (difficulty 4 or 5) fall on the same day, a warning icon appears. This helps admins avoid overloading players!

💡

Hover over a challenge block on the calendar to see a quick popup with its name, type, and difficulty.

🏆 Events

Events are big, exciting server-wide activities — bigger than regular challenges. All players participate at the same time, and a leaderboard tracks who's winning!

🎮 Event Types

🐉

Boss Raid

A giant boss monster spawns. Everyone teams up to defeat it! The boss has multiple health phases and special attacks.

🗺️

Treasure Hunt

Chests are hidden across a region. Players race to find and loot them all before time runs out.

🏗️

Build Battle

Players get their own plots and build on a theme. Admins (or the system) judge who built the best!

⚔️

Clan War

Two clans battle each other. The war result is automatically tracked and recorded. (These events are created automatically by the clan system.)

📋 Event States

StateMeaning
UPCOMINGScheduled for the future, not started yet
ACTIVEHappening right now — players can join!
FINISHEDOver — results and leaderboard are locked in

➕ Creating an Event SUPER_ADMIN

  1. Go to the Events page and click New Event.

  2. Choose the event type. The form changes based on type — Boss Raid asks for the boss's name, HP, and arena coordinates; Treasure Hunt asks for how many chests and a loot preset; Build Battle asks for a theme and plot size.

  3. Give the event a title and pick a scheduled start time.

  4. Click Create. The event is now in UPCOMING state.

▶️ Starting & Ending an Event

On the Events page, upcoming events show a Launch Now button. Active events show an End Event button. You can also use these buttons from the Event Detail page.

ℹ️

Active events auto-refresh their leaderboard every 5 seconds so you can watch scores update in real time!

🏅 Leaderboard

Every event has a live leaderboard. The top 3 players get medal icons (🥇🥈🥉). The leaderboard is visible to all logged-in users and updates automatically while the event is active.

🎁 Rewards

Rewards are prizes players earn for completing challenges, participating in events, or given manually by admins. You create reward templates first (what the reward is), then they can be granted to players.

🌟 Reward Types

TypeWhat the Player Gets
ITEMA specific Minecraft item (sword, armour, blocks, etc.)
XPExperience points that level them up in-game
CURRENCYCoins they can spend on cosmetics or in the marketplace
COMMANDRuns any server command (useful for giving ranks, permissions, etc.)
MYSTERY_BOXA random reward chosen from a weighted loot table — the player doesn't know what they'll get!

✨ Rarity Levels

Rewards have a rarity that affects how the card looks in the UI (colour-coded):

⚪ Common

Everyday items. Easy to get, given out frequently.

🔵 Rare

Better items, less common. Something to aim for!

🟣 Epic

Special prizes for tough challenges or major events.

🟡 Legendary

The best of the best. Very rare, very valuable!

🎲 Mystery Box Loot Tables

When you create a MYSTERY_BOX reward, you build a loot table — a list of possible prizes, each with a weight (percentage chance). All weights must add up to exactly 100.

Example loot table:

PrizeWeightChance
100 Coins6060% chance
Diamond Sword2525% chance
VIP Rank (COMMAND)1515% chance
⚠️

Weights must total exactly 100. If they don't, the system will reject the reward with an error message.

➕ Creating a Reward SUPER_ADMIN

  1. Go to Rewards and click New Reward.

  2. Give it a name (e.g. "Diamond Bundle"), choose the type and rarity.

  3. Fill in the type-specific config fields — for ITEM, pick the item material and quantity; for XP, enter the amount; for COMMAND, write the command template; for MYSTERY_BOX, add your loot table entries.

  4. Click Create. The reward template is now available to assign to challenges or grant manually.

⚔️ Clans

Clans are player groups — like teams or guilds. Players form them in-game, earn XP together, and battle other clans in wars. Admins can view all clans and their history from the dashboard.

📊 Clan Info

Each clan has:

  • Name & Tag — the clan's full name and a short tag shown in chat (e.g. [EPIC])
  • Level — increases every 1,000 XP earned, maximum level 10
  • XP — earned by completing challenges, winning wars, etc.
  • Members — list of players and their role (Leader, Officer, Member)
  • Home — a saved location that members can teleport to
  • War History — record of past clan wars (wins, losses, draws)

📋 Clan Roles

🏹

⛏️

🗣️ NPCs

NPCs (Non-Player Characters) are custom characters that stand in specific spots in the Minecraft world. When a player right-clicks one, it talks to them! Admins create and manage NPCs from the dashboard.

🤖 NPC Types

📖

Guide

Gives players helpful tips and information about the server. Perfect for the spawn area.

❓

Quest Giver

Offers quests to players. The NPC remembers which quests the player has completed and unlocks new ones step by step.

🛍️

Merchant

A shop NPC where players can buy or trade items.

⭐ Quest Giver NPC — How Quests Unlock

Quest Giver NPCs use a tier system. If an NPC has 3 quests assigned, a new player starts at tier 0 and can only see the first quest. Once they complete it, tier becomes 1 and the second quest unlocks. And so on!

Quest StatusHow It Looks in Chat
Already completed✓ quest-name (completed)
Available now➤ quest-name (available)
Still locked🔒 quest-name (locked — complete previous quest first)

➕ Creating an NPC SUPER_ADMIN

  1. Go to the NPCs page and click New NPC.

  2. Enter the NPC's name and title (displayed above their head).

  3. Choose the type (Guide, Quest Giver, or Merchant).

  4. Enter the world name and X, Y, Z coordinates where they should stand.

  5. Add dialogue lines — each line on a new line. These are what the NPC says when clicked.

  6. For Quest Givers, add Quest IDs — one per line, in the order you want them to unlock.

  7. Click Save. The plugin will automatically spawn the NPC in-game within seconds.

🔄 Dynamic Dialogue

Some NPCs have dynamic dialogue — if there's a live event happening, the NPC automatically mentions it! For example, a Guide NPC might say "There's a Boss Raid starting in 5 minutes at the arena!" when a Boss Raid event is upcoming.

🛡️ Moderation

The Moderation section is where staff handle player reports, review chat, and keep the server safe and friendly.

Access: SUPER_ADMIN MODERATOR

📨 Reports Tab

When a player reports someone in-game (using /report <player> <reason>), it appears here. Each report shows:

  • Who filed the report and who was reported
  • The reason given
  • A chat snapshot — the last few messages from that player so you can see context
  • The current status of the report

Report Statuses

StatusMeaning

Actions on a Report

When viewing a report, you can:

  • Mark Reviewed — signals you've seen it MODERATOR
  • Resolve — close the report as handled MODERATOR
  • Mute 1 Hour — temporarily silence the reported player MODERATOR
  • Kick — remove the player from the server (they can rejoin) MODERATOR
  • Ban — permanently (or temporarily) block the player SUPER_ADMIN only

💬 Chat Log Tab

Every message sent in chat is saved here. Staff can search the chat log to investigate players:

  • Search by message content — find anyone who said a specific word
  • Search by player name — see everything one player has said
  • Date filter — narrow down to a specific time period
  • Flagged messages — messages that were automatically flagged by the chat filter appear with a 🚩 icon

📋 Audit Log Tab

The Audit Log is a read-only permanent record of every moderation action ever taken on the server (mutes, kicks, bans, unmutes, unbans). You can filter by player to see their full moderation history. Nothing can be deleted from here — it's a tamper-proof log.

🎮 In-Game — Challenges

Players don't need to use the website for challenges! Everything happens automatically in-game. Here's how it works from a player's perspective:

  1. A challenge becomes active (e.g. "Mine 100 diamonds"). The player doesn't need to do anything to sign up.

  2. As the player mines diamonds, their progress increments automatically. No commands needed!

  3. When they hit 100 diamonds, a notification appears in chat saying the challenge is complete.

  4. The reward is delivered instantly. If the player is offline when the reward processes, it queues and delivers the next time they log in.

📋 /challenges

Usage: /challenges [page]

Lists all active challenges with your personal progress bar for each one. Results are paginated (5 per page) — use /challenges 2 to go to page 2. Only usable by players in-game (not from the server console).

🔄 How Things Work — Player Join

Here's exactly what happens every time a player logs into the Minecraft server:

  1. Minecraft fires a "Player Joined" event. The GreeterPlugin catches it.

  2. The plugin calls the web API to register or update the player's profile (increments join count, updates last seen time).

  3. The plugin checks if there are any queued rewards waiting for this player (from challenges completed while they were offline). If so, they're delivered now.

  4. The plugin checks if any of the player's reports have been resolved since they last logged in. If so, they get a notification.

  5. A welcome message may be broadcast based on the DAILY_LOGIN trigger configuration.

🎁 How Things Work — Reward Delivery

When a reward is granted (automatically or by an admin), here's the delivery process:

  1. The system checks if the player is currently online.

  2. If online: The reward is delivered immediately. Items are put in their inventory, XP is added, commands run, coins are credited.

  3. If offline: The reward is saved to a queue in the database.

  4. Next time the player logs in, the GreeterPlugin checks the queue and delivers all pending rewards right away.

Mystery Box Resolution

When a MYSTERY_BOX is delivered, the plugin rolls the weighted loot table (like rolling a weighted dice), picks one prize, and delivers that prize instead. The player sees a message telling them what they got!

🏆 How Things Work — Event Lifecycle

  1. An admin creates an event on the dashboard with a scheduled start time.

  2. The EventPlugin polls the API every 35 minutes for upcoming events. When an event is less than 35 minutes away, the plugin starts preparing (arena setup, NPC announcements).

  3. The admin clicks Launch Now on the dashboard. The event switches to ACTIVE.

  4. The EventPlugin starts the game logic for that event type (spawns the boss, scatters treasure chests, creates build plots).

  5. As players participate, scores are tracked. The dashboard leaderboard updates every 5 seconds.

  6. When the event ends (admin clicks End or the timer runs out), the plugin sends the final leaderboard to the API and the event becomes FINISHED.

  7. Top players receive any configured rewards automatically.

🛡️ How Things Work — Moderation

  1. A player uses /report Steve Being rude in-game.

  2. The ModerationPlugin captures the last 10 chat messages from Steve as a snapshot and sends the report + snapshot to the API.

  3. The report appears in the dashboard with PENDING status.

  4. If Steve gets 2 more reports in the next 24 hours, all his pending reports jump to ESCALATED.

  5. A moderator opens the report, reads the chat snapshot, and decides to mute Steve for 1 hour.

  6. The mute is logged to the Audit Log. Steve gets a mute applied in-game immediately. The report is marked RESOLVED.

  7. Next time the reporting player logs in, they receive a notification: "Your report against Steve has been resolved."

❓ FAQ

A player says they didn't receive their reward — what do I check?

Go to Players → [Player Name] → Reward History tab. If the reward shows there, it was recorded but maybe the in-game delivery glitched. Check if they were online at the time. Rewards queue automatically if the player is offline and deliver on next login — so if they just need to log in again, tell them to do that!

How do I stop a challenge from running?

Go to Challenges → [Challenge Name] → Edit and set the end date to now (or a past date). The challenge will immediately stop being active and no further progress will be tracked.

Can I delete a reward that's been given to players already?

Yes, but only if the reward isn't linked to an active challenge. Go to the reward's detail page and click Delete. Past grant records are kept in the Reward History for each player — deleting the template doesn't erase those records.

What happens to a clan if the leader leaves?

Two cases: (1) If the leader is the only member left, the clan is automatically disbanded and deleted. (2) If there are other members, the system picks the next most senior active member (an Officer first, otherwise the longest-standing Member) and makes them the new Leader.

A player has a title equipped but it's not showing in chat. Why?

Check that the title still exists in the Cosmetics section (it may have been deleted). If the title exists, the issue may be a plugin restart — the cosmetics plugin caches profiles, and on a fresh join the title should re-appear. Ask the player to relog.

How do I give someone moderator access to the dashboard?

Account roles are set directly in the database by a SUPER_ADMIN. There is no in-dashboard role editor — ask a developer or server owner to run an update on the account's role field in the database, or use the seed / user-creation tools available to server owners.

Can I schedule an event to start automatically without clicking Launch Now?

Currently the event switches to ACTIVE only when an admin clicks Launch Now on the dashboard or in the Event Detail page. The scheduled time on an event is informational (it helps the plugin pre-load the event), but the final "start" signal still requires a manual admin action.

The chat log shows 500 error — why?

If the Minecraft plugin is sending malformed chat log data (missing required fields like timestamp or playerId), the API will return an error. Check plugin logs for JSON formatting errors. The server's error middleware now returns 400 for bad data, so if you still see 500, it may be a database connectivity issue — check the server's health on the Dashboard page first.
